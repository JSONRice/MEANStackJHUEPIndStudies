NodeJS setup really depends on what your project setup entails, but there are a few base standards that go along with node. To start using NodeJS download it from their web site (Google search: 'Node download') then once installed you will have NPM (Node Package Manager). Head over to the root level of the web application in the terminal (command line) and run the following to download the dependencies:

npm install

If you suspect any dependencies are out of date in the web app then you can update them as follows:

npm update

To view the current version of a dependency such as karma type:

npm list karma

Also if you just want to update karma try:

npme update karma

The dependencies are all stored within the package.json at the root level of the web application.

Checkout the package.json for a list of production and developer level dependencies.

Checkout the app.js at the root level. The app.js file is the bootstrap file Node loads all the dependencies and connects the app to the network and database by caling this file. To call file in Node simply type:

node app.js

Since the app.js requires a Mongo database to run, you will need to startup your Mongo instance with a database location. Read up on the Mongo slides for more info.
